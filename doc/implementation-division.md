## 背景

当前项目技术栈固定为 `OpenTUI + React`，目标是基于 React 的熟悉模式构建公司自己的 CLI 产品能力。CLI 产品核心是“交互 + 状态管理 + 工作流”。

## 分工方案

1. 助手负责基础架构与可扩展能力

- 搭建 OpenTUI + React 的 App Shell（如会话区、输入区、状态栏）。
- 设计并实现可扩展接口层：Provider、Tool、Session、Config。
- 提供稳定的数据流与状态管理骨架（保证可迭代）。
- 复杂 TypeScript 类型由助手优先兜底实现。

2. 你负责交互体验与产品表达

- 定义交互流程（命令流、输入反馈、错误提示、快捷键体验）。
- 主导 UI 结构与文案风格（组件布局、信息层级、可读性）。
- 按业务优先级迭代界面细节与产品感。

3. 协作方式

- 助手先给出可运行骨架，你在此基础上逐步改 UI 与交互。
- 你在修改中遇到 TS 问题，助手负责解释并补齐类型实现。
- 每次迭代保持“小步可运行”：先能跑，再增强。

## TypeScript 学习最小范围（第一版够用）

1. `type` / `interface`
2. 函数参数与返回值标注
3. `Promise<T>`
4. 联合类型 `A | B`
5. 可选属性 `foo?: string`

## 约束

- 不移除 OpenTUI 技术栈。
- 不改为纯 Node 命令式 CLI（除非你明确要求）。
- 技术问题优先查官方文档与仓库（可通过 Context7 辅助）。
